## Project Overview

**DocBel** is a Next.js 16.2.4 single-page application providing Belgian citizens with a portal for administrative documents, tools, calculators, and resources related to unemployment, CPAS assistance, and employment services.

The app is an **SPA** (Single Page Application) — all navigation is client-side state management, no file-based routing for main views.

### Tech Stack
- **Framework**: Next.js 16.2.4 (App Router with Server/Client components)
- **UI Library**: shadcn/ui v4.4.0 (component library built on Radix UI + Tailwind)
- **Styling**: Tailwind CSS 4.2.4 + inline styles for dynamic color theming
- **Icons**: Lucide React (shadcn components) + 15 custom SVG icons (in `components/docbel/icons.tsx`)
- **Language**: TypeScript 5.9.3 (strict mode)
- **Font**: Plus Jakarta Sans (300, 400, 500, 600, 700, 800 weights from Google Fonts)
- **Colors**: Dual palette system (DARK_COLORS / LIGHT_COLORS) with 9 properties each, defined in `lib/docbel-data.ts`
- **State Management**: Local client-side `useState` (no Redux, Zustand, or Context — keeps bundle small)

## Data Model (Single Source of Truth: `lib/docbel-data.ts`)

### Tool Structure & Rendering
25 tools across 6 categories. Each tool has:
```typescript
{
  id: string                  // Unique identifier
  icon: React.ReactNode       // Lucide or custom SVG icon
  title: string               // Display name
  desc: string                // One-line description
  category: string            // "Documents" | "Calculators" | "Locators" | etc.
  type: string                // Sub-view routing type (see below)
  popular: boolean            // Show popular badge on cards
  timeMin: number             // Estimated time in minutes
  color?: string              // Optional accent color for hero/cards
  // Type-specific data (passed to corresponding sub-view component)
  ...typeSpecificProps
}
```

### Tool Types & Sub-View Routing
Tool `type` field determines which component in `tool-views.tsx` renders. 8 types:

1. **"calc-preavis"** → `CalcPreavis`
   - Notice period calculator (Belgian law 2013)
   - Inputs: Years of seniority (20+ brackets), employment status (CDI/CDD/Intérim)
   - Output: Notice period in days

2. **"calc-agr"** → `CalcAGR`
   - Guaranteed Income Allocation calculator
   - Inputs: Salary, hours worked
   - Output: AGR entitlement calculation

3. **"calc-cp"** → `CalcCP`
   - Minimum wage lookup by sector
   - Dropdown: 12 sectors (Horeca, Commerce, Construction, etc.)
   - Displays current minimum wage per sector

4. **"locator"** → `Locator`
   - Bureau address finder (ONEM offices or payment organisms)
   - Radio buttons: ONEM or payment organisms
   - Displays 6/4 addresses with tel/web contact info

5. **"tutorial"** → `Tutorial`
   - Step-by-step guides (4 steps per tool)
   - Progress bar and Previous/Next navigation
   - Step titles and descriptions from data

6. **"info"** → `InfoPanel`
   - FAQ accordion sections for ALE and CPAS
   - Title/body pairs with expand/collapse

7. **"links"** → `LinkPanel`
   - Links to external Belgian services
   - Actiris, VDAB, FOREM, ADG with descriptions and icons

8. **"form"** → `FormFlow`
   - Generic 3-step form workflow: Fill → Preview → Download PDF
   - State management for form inputs and submission
   - PDF download placeholder (TODO)

### Color System
**Dual theme approach**: Two complete palettes with 9 CSS properties each
- `DARK_COLORS` (theme when `dark = true`)
- `LIGHT_COLORS` (theme when `dark = false`)

Properties in each palette:
- `bg` — Main page/section background
- `text` — Primary text color
- `text_secondary` — Secondary text (labels, hints)
- `border` — Border and divider color
- `hover_bg` — Hover state background
- `icon_bg` — Icon container background
- `card_bg` — Card/panel background
- `card_text` — Text within cards
- `accent` — Interactive elements (buttons, highlights)

**Why inline styles?** Tailwind's color system is compile-time. Real-time dark/light switching requires runtime styles. All DocBel components receive `colors` prop:
```typescript
style={{ backgroundColor: colors.card_bg, color: colors.card_text }}
```

**Accent color**: Fixed at `#C8102E` (Belgian red tricolor) in `app/page.tsx`. Planned for dynamic customization via TweaksPanel feature.

## Styling Guidelines

### Tailwind + Inline Styles (Hybrid Approach)

**Tailwind** (layout, spacing, sizing, animations):
```typescript

className="flex items-center justify-between gap-2"
```

**Inline styles** (dynamic colors only):

```typescript
  backgroundColor: dark ? colors.DARK_COLORS.bg : colors.LIGHT_COLORS.bg,
  color: dark ? colors.DARK_COLORS.text : colors.LIGHT_COLORS.text
}}
```

**Why?** Tailwind is compile-time. Real-time theme switching requires runtime styles.

- Use class composition: `className="flex gap-4 p-6"`

## 🚨 Common Pitfalls & Solutions (IMPORTANT - Prevents Repeated Mistakes)

### 1. Nested Button Errors with DropdownMenuTrigger

**Problem**: Using `asChild` with a `Button` child in `DropdownMenuTrigger` creates nested buttons:
```tsx
// ❌ WRONG - Creates nested <button> elements
<DropdownMenuTrigger asChild>
  <Button variant="ghost"><MoreHorizontal /></Button>
</DropdownMenuTrigger>

// ❌ ALSO WRONG - asChild with div causes React error about unrecognized prop
<DropdownMenuTrigger asChild>
  <div role="button"><MoreHorizontal /></div>
</DropdownMenuTrigger>
```

**Solution**: Remove `asChild` entirely and style the DropdownMenuTrigger button directly:
```tsx
// ✅ CORRECT - DropdownMenuTrigger renders as button with inline styles
<DropdownMenuTrigger className="h-8 w-8 p-0 hover:bg-gray-100 rounded-md transition-colors inline-flex items-center justify-center">
  <MoreHorizontal className="w-4 h-4" />
</DropdownMenuTrigger>
```

**Key Rule**: `DropdownMenuTrigger` **already renders a button**. Do NOT add `asChild` unless you need a different element type. If using `asChild`, use a non-button element like `<div>` with proper `role="button"` and keyboard support.

### 2. Spacing Standards (margin, padding, gap)

**Consistent Hierarchy**:
- **Main page container**: `px-6 py-6` (horizontal and vertical padding for full-width sections)
- **Between major sections**: `gap-8` (Tailwind's `space-y-8`)
- **Cards within grids**: `gap-4` (medium spacing between grouped items)
- **Items within cards**: `gap-3` (tight spacing for list items)
- **Card headers**: `pb-3` padding below header title

**Example Structure**:
```tsx
// Page container
<div className="space-y-6 px-6 py-6">
  {/* Header section */}
  <div className="gap-4">...</div>
  
  {/* Stats cards grid */}
  <div className="grid grid-cols-1 md:grid-cols-4 gap-4">
    {/* Each card has internal gap-3 */}
  </div>
  
  {/* Main content */}
  <Card className="gap-4">
    <CardHeader className="pb-3">Title</CardHeader>
    <CardContent className="gap-3">
      {/* Items with gap-3 */}
    </CardContent>
  </Card>
</div>
```

**Never**:
- Leave main page containers without padding (`px-6 py-6`)
- Mix `gap-` and `space-` classes without understanding difference
- Use inconsistent spacing values (don't jump from `gap-2` to `gap-8`)

### 3. UTF-8 Encoding in API Responses

**Problem**: Special characters (à, é, ô, ç) appear corrupted in database or API responses

**Solution**: Always add `charset=utf-8` to API response headers:
```tsx
// In all API routes (GET, POST, PATCH, DELETE)
return NextResponse.json(
  { data: ... },
  { headers: { 'Content-Type': 'application/json; charset=utf-8' } }
);
```

**Also Important**: 
- If database has corrupted data, create a reset endpoint to reinitialize with proper UTF-8 encoded data
- Test with French characters: "Mise à jour", "Réforme", "CPAS"

### 4. Admin Page Component Patterns

**For dropdown/action menus**:
- Use plain `DropdownMenuTrigger` without `asChild` for button-like triggers
- Use `asChild` only when you need non-button elements AND include proper `role="button"` and `tabIndex={0}`

**For tables with row actions**:
- Apply consistent padding: `px-6 py-6` on container
- Keep action column right-aligned: `className="text-right"`
- Use ghost buttons for secondary actions
- Test all dropdown interactions work without console errors

**For filter sections**:
- Label every filter: `<label className="text-sm font-medium block mb-2">Label</label>`
- Use consistent input spacing
- Group related filters in flex containers with `gap-4`

### 5. Console Error Checklist

When building UI components, verify:
- [ ] No hydration errors (server/client mismatch)
- [ ] No "React does not recognize..." warnings about DOM props
- [ ] No nested button errors in dropdowns
- [ ] No uncaught errors in console
- [ ] All interactions (clicks, hovers) work smoothly

**Quick Test**: Open browser DevTools console, refresh page, check for red errors.
