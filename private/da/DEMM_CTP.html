Nature de demande CTP                                                                       => création le 24/01/2020

  

Ø  Résumé

A partir du 01/01/15, un travailleur, qui continue le travail au-delà de l’âge de la pension (65 ans), peut bénéficier des allocations de chômage temporaire

=> exception : chômage temporaire pour cause de force majeure médicale

=> cependant, les allocations en CT ne peuvent pas être cumulées avec la pension

 

A partir du mois qui suit ses 65 ans et dès le premier mois en chômage temporaire,

nous sommes obligé d’introduire une demande d’allocations.

=> il s’agit normalement d’une démarche purement technique afin de relancer une intégration

auprès de nos systèmes mais également auprès de l’ONEM.

à l'intégration "habituelle" se termine au plus tard le dernier jour du mois

dans lequel une personne atteint 65 ans

ou à une date antérieure (par exemple en raison de l'archivage, transfert,…)

à l'intégration est alors demandée jusqu’à la date limite du dernier jour du mois

au cours duquel la personne atteint 99 ans.

à nous récupérons l'intégration avec le code 15

 

Lors de cette demande CTP, il existe toutefois 3 possibilités :

1) la demande CTP coïncide avec une demande obligatoire                                à scénario 1

à ce cas de figure est plus fréquent pour la demande CTP

car préfixes possibles avec du chômage temporaire avant 65 ans

(02, 04, 05, 06, 57, 58)

à à partir de 65 ans, il ne reste que les préfixes 02 et 04

2) la demande CTP coïncide avec une demande facultative                                 à scénario 2

3) la demande CTP est simplement constituée pour une nouvelle intégration        à scénario 3

Ø  Méthode de travail informatisée

 

Scénario 1 (CTP avec une DA obligatoire)

 

à suivre la méthode informatisée décrite sur la fiche HELP sur la nature INT

(demande obligatoire)

 

+ faire et vérifier l’intégration via l’écran « Modification signalétique membre » (ML010/03)

=> vérifier les informations 

=> ENTER pour l’intégration

=> vérifier que l’intégration est ok via fenêtre F2)

et imprimer la C1 écran « composition familiale » (AF010/01)

=> vérifier la composition familiale

=> F15 : impression C1

 

 

 

 

 

Scénario 2 (CTP avec une DA pour révision du code chiffré à la hausse)

 

à suivre la méthode informatisée décrite sur la fiche HELP sur la nature INT

(demande facultative)

 

+ faire et vérifier l’intégration via l’écran « Modification signalétique membre » (ML010/03)

=> vérifier les informations 

=> ENTER pour l’intégration

=> vérifier que l’intégration est ok via fenêtre F2)

  et imprimer la C1 via écran « composition familiale » (AF010/01)

=> vérifier la composition familiale

=> F15 : impression C1

 

Scénario 3 (CTP uniquement pour l’intégration)

 

® vérifier l’article d’admissibilité valable lors de la dernière demande en chômage temporaire

Art. admis.

Descriptions

42K30

prouve les conditions d’admissibilité sur base de l’article 30

42K33

prouve les conditions d’admissibilité sur base de l’article 33

42L36

prouve les conditions d’admissibilité sur base de l’article 36

42J30

bénéficie de la mesure transitoire avec un statut temps plein

42J33

bénéficie de la mesure transitoire avec un statut temps partiel

42T99

admissible art. 42 (uniquement pour CT) suite exclusion procédure DISPO

42M99

n’est pas admissible (pas droit au CT avec codes 51 et 511)

 

1er) écran « sélection demande » (AC040/01) : faire F6 pour création

=> date de demande : 1er du mois qui suit le mois de ses 65 ans 

=> nature demande : CTP

=> préfixe chômage : 02 ou 04

=> délai d’introduction : pas de délai

 

2ème) écran « Modification signalétique membre » (ML010/03)

=> vérifier les informations

=> ENTER pour l’intégration

à vérifier que l’intégration est ok via fenêtre F2

 

3ème) écran « demande de paramètres » (AE035/03)

=> compléter N sur les demandes de flux sauf en fonction de la situation

=> recopier l’article d’admissibilité valable lors de la dernière demande

 

4ème) écran « documents externes » (AE110/06)

=> C3.2 EL automatiquement réclamé par le programme

     + éventuellement DMFA en fonction du type d’admissibilité + autres flux

 

5ème) écran « composition familiale » (AF010/01)

=> vérifier la composition familiale

=> F15 : impression C1

 

6ème) écran « liste des émargements » (AE010/01)

=> F20 pour accéder à l’écran « demande/réception des documents externes »

 

7ème) écran « demande/réception des documents externes » (AE110/01)

=> option 1 devant le WECH 502 (C32) ou 505 (C32PAI)

+ ENTER pour importer l’occupation

=> plusieurs écrans concernant l’activité

à zone QX4 : effacer les données sur les 2 lignes pour alimenter le compteur

à ENTER plusieurs fois jusque à l’écran « émargements »

           

 

8ème) retour à l’écran « liste des émargements » (AE010/01)

=> Enter

 

9ème) écran « demande d’allocations » (AG 010/03)

=> état demande : 1 (ou F4 sur la zone pour obtenir un autre choix)

=> zone « C2 » : Y

=> ajouter quelques remarques (conseillé pour attirer l’attention)

=> F20 pour rappeler la dernière codification valable en chômage temporaire

 

10ème) écran « modification codification – C2 » (CS010/05)

=> méthode de calcul 08 affichée automatiquement car pas concerné par dégressivité

=> barème (1 seule ligne) s’affiche automatiquement : pas de F20 à faire
