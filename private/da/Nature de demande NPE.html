Nature de demande NPE                                                            è mise à jour le 03/12/2019

 

Ø Réglementation

Art. 116 de l’AR du 25/11/1991

 

Ø Résumé

Demande utilisée lorsqu’un chômeur demande à nouveau les allocations de chômage complet (ou assimilé) après une interruption suite à une reprise de travail ou un autre événement

qui lui permet d’obtenir :

* soit, un retour en première période (uniquement si reprise de travail salarié) (art. 116§1)

* soit, une prolongation des périodes d’indemnisation (art. 116§2)

=> exemple : DA après 1 an de travail salarié temps plein è retour en 1ère période

 

Cette nature de demande est également utilisée

pour le secteur HORECA (art. 116§3) et les artistes (art. 116§5)

 

!!! Ne pas oublier de vérifier si le chômeur respecte toujours les conditions d’admissibilité.

=> pour chaque demande, on l’on peut appliquer l’article 42 : vérifier

* en 1er, si retour en 1ère période possible

* en 2ème, si prolongation période possible

* également si le code chiffré doit être revu

=> !!! si les conditions de révision du code chiffré sont également remplies

(interruption du chômage ≥ 24 mois et reprise occupation salariée ≥ 4 semaines ininterrompues chez le même employeur), vous devez utiliser la nature de demande NCP

Ø Formulaire à introduire

C1 et REGIS si nécessaire (interruption chômage ≥ 1 an ou modification des données)

+ C4 + annexe éventuellement + DMFA obligatoire (+ autres documents si nécessaire)

 

Ø Formulaire de paiement :

C3A ou autre selon les cas + IDE service placement (sauf si dispensé)

 

Ø Méthode de travail informatisée

 

1) écran « sélection demande » (AC040/01) : faire F6 pour création

=> date de demande : date à laquelle le chômeur demande à nouveau les allocations

=> nature demande : NPE

=> type chômage : 01, 03, 09, 10, 11, 18,…

=> délai d’introduction : 2 mois de date à date

 

2) écran « signalétique membre » (ML010/03)

=> modifier les données si nécessaire

 

3) écran « demande de paramètres » (AE035/03)

=> vérifier le dernier jour indemnisé en chômage complet, ainsi que la date

   de dernière demande (éventuellement, à contrôler dans une autre régionale)

=> compléter Y ou N concernant la demande de documents électroniques

   (flux indépendant, flux interruption de carrière, C6, etc.)

=> compléter l’article (F4) : 161 (retour) ou 162 (prolongation)

=> en cas de retour, compléter le niveau (F4) : 1, 2 ou 3 en fonction des conditions de retour

=> contrôler la période de référence (possibilité de la déplacer en cas de retour)

 

4) écran « documents externes » (AE110/06)

=> le programme demande automatiquement la DMFA

+ éventuellement d’autres flux en fonction des Y ou N dans l’écran précédent

=> si besoin de flux électroniques non réclamés par le programme 

       àF6 pour demande manuelle

 

5) écran « composition familiale – REGIS (AF030/01)

=> comparaison des données avec celles du REGIS                                                      

(adresse, nationalité et composition familiale)

à les données doivent être identiques

à en cas discordance maintenue : encodage d’un code pour justifier la différence

à impression automatique du C1 annexe REGIS à la fin du dossier

 

6) écran « composition familiale » (AF010/05)

 => F15 pour imprimer C1

 

7) écran « liste des émargements » (AE010/01)

=> encoder la nature de l’interruption (F6)

=> encoder données pour trimestres manquants DMFA si nécessaire (via annexe)

à obligatoire dans le cas de la prolongation pour avoir l’effet au niveau de la Time Line

8) écran « demande d’allocations » (AG010/03)

=> F6 pour accéder à la Time Line => contrôler le résultat

à si des modifications sont nécessaires (ex. : déplacement PR, modification niveau),

   il faut retourner à l’écran « demande de paramètres » pour y adapter les données. 

 à ensuite, retour à l’écran « demande d’allocations »

      et F6 pour générer une nouvelle Time Line.

 

=> état de la demande : valeur 1 (si complet ou F4 sur la zone)

=> zone C2 : Y

=> codification F21 : type chômage + code chiffré (reste le même)

=> mettre un BL en cas de situation douteuse ou litigieuse

=> zone « type » = 161 ou 162 encodage automatique

   suite article encodé dans écran « demande paramètre »

=> dans le cas d’une prolongation : zones « prol. » et « db. Occ. » 

      à encoder nombre de mois complet

        + date à laquelle débute le 1er événement prolongeable

=> ajouter commentaires éventuels    

   

9) écran « modification codification – C2 » p.1

=> méthode de calcul et autres données en fonction du dossier

   (voir fiches help méthodes de calcul)

à d’office 01 si art. 161 retour en 1ère période

=> F20 pour calculer barèmes 

 

10) écran « modification codification – C2 » : page 2 via F8

=> adapter le compteur vacances et le facteur J s’il y a lieu

(effectuer d’abord l’étape 11 si vacances pas encore calculées)

  

11) écran « calcul vacances » (AC200/01)  

=> calculer le facteur J si pas encore fait => F20 sur l’écran

   (attention d’avoir l’émargement complet sans zone en surbrillance)
